# Ontology POC — EM Planning Layer

This is **my** planning and tracking layer for the Ontology Platform POC.
It is not the team's working repo. The team uses Jira for tracking and Boeing GitLab for code artifacts.

## What lives here
- `roadmap/` — the 7-week plan, milestones, and current status
- `meetings/` — meeting notes (kickoff, weekly syncs, stakeholder check-ins)
- `decisions/` — decision log (what we decided, why, who pushed back)
- `risks/` — risk register, updated weekly
- `stakeholders/` — stakeholder map, what each one cares about, last-touch dates
- `weekly/` — my own weekly review notes

## What does NOT live here
- Team task tracking → Jira
- Code, schemas, technical artifacts → Boeing GitLab
- Anything sensitive that shouldn't be on personal infra (this is `em-playbook`-adjacent, treat it as such)

## How I use it
- Open the repo on Mondays for the weekly review.
- Update during/after meetings, not "later."
- If a file hasn't been touched in 3 weeks, it either gets updated or deleted. No zombie docs.

## Discipline note
This repo will fail if I treat it as a tool I have to feed. It works only if it's the place I think *out loud* about the POC. The moment it becomes a chore, archive it.# ontology-poc
